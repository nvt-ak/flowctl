## 1. Methods by phase (expand as needed)

| Phase | Method | Typical duration | Output |
|-------|--------|------------------|--------|
| Discovery | Stakeholder interview | 30–60 min | Pain themes, constraints |
| Discovery | Competitive scan | 1–3 h | Feature / UX matrix |
| Definition | Persona | 1–2 h | Persona one-pager |
| Definition | Journey map | 2–4 h | Stages, emotions, system touchpoints |
| Design | Wireframe / prototype | half day+ | Testable artifact |
| Validation | Usability test | ~1 h / participant | Issues ranked by severity |

Pick the **minimum** set that answers the open design questions — do not run every method by default.

## 3. User flow notation

Document:

- Entry points and preconditions.
- **Happy path** to success.
- **Error** and empty states with recovery.
- Decision branches with clear labels.

ASCII or diagram tool — consistency matters more than format.
